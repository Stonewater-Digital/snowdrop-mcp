### Use Case 1: The Automated Fund Accounting Audit
**The Problem:** You have 500 rows of unstructured ledger entries and need a reconciled trial balance before the NY session opens.
**The Agent Workflow:**
1. Agent uses `reconcile_bank_statements` to match the raw CSV against on-chain wallet feeds.
2. Agent calls `calculate_daily_nav` to price the current token assets.
3. Agent uses `generate_journal_entries` to properly categorize the discrepancies into double-entry accounting format.
4. Agent calls `export_trial_balance_xbrl` to output the final, audited report ready for regulatory filing.

### Use Case 2: Zero-Day Regulatory Compliance Check
**The Problem:** A new SEC ruling on crypto staking drops. Does your firm's current operational strategy violate the new framework?
**The Agent Workflow:**
1. Agent triggers `monitor_sec_filings_and_press` to pull the exact text of the new regulation.
2. Agent runs `run_mica_compliance_audit` or `run_sfd_compliance_check` against your firm's smart contract deployment logs.
3. Agent cross-references `crypto_tax_liability_estimator` to see if the new ruling triggers a taxable event based on your historical yield.
4. Agent generates an executive summary outlining your exposure and suggested structural fixes.

### Use Case 3: The Cross-Chain Treasury Manager
**The Problem:** You have fragmented liquidity across Ethereum, Solana, and TON, and need to rebalance while minimizing slippage and avoiding high gas fees.
**The Agent Workflow:**
1. Agent uses `fetch_yield_opportunities` to scan current lending rates on Aave, Kamino, and TON pools.
2. Agent calls `calculate_cross_chain_arbitrage` to evaluate bridge fees versus expected yield.
3. Agent runs a `portfolio_stress_test_var` to ensure the rebalancing doesn't expose the treasury to outsized smart contract risk.
4. Agent drafts the specific transaction payloads required for the human-in-the-loop to sign and execute.
