## What This Repo Is

A self-hostable MCP server (Community Edition) with 1,500+ financial, compliance, DeFi, and infrastructure skills. The server runs in two modes:

- **HTTP mode** (`PORT=8000 python mcp_server.py`) — FastAPI wrapper with `/health`, `/.well-known/agent.json`, and `/mcp` endpoints
- **Stdio mode** (`python mcp_server.py`) — for local MCP clients like Claude Code desktop and Cursor

Skills are auto-synced from the private `snowdrop-core` repo on every deploy. Premium skills live only in `snowdrop-core` — this repo contains free skills and auto-generated premium stubs.

## Running the Server

```bash
# HTTP mode
PORT=8000 python mcp_server.py

# Stdio mode (for Claude Code, Cursor)
python mcp_server.py
```

Health check: `curl http://localhost:8000/health`
Agent card: `curl http://localhost:8000/.well-known/agent.json`

## Adding a Skill

1. Create or find the right category directory under `skills/`
2. Add your Python file with this pattern:

```python
TOOL_META = {
    "name": "your_skill_name",
    "description": "One sentence description.",
    "parameters": {
        "param_one": {"type": "str", "description": "What it is"},
    }
}

def your_skill_name(param_one: str) -> dict:
    """Executive Summary: what this skill does."""
    try:
        # your logic here
        return {"status": "ok", "data": {}, "timestamp": ...}
    except Exception as e:
        return {"status": "error", "error": str(e), "timestamp": ...}
```

3. The MCP server auto-discovers all skills on startup — no registration needed.
4. Do **not** use `**kwargs` in skill signatures — FastMCP 3.x rejects them.

## Premium Skill Policy

| Tier | What it means | Where code lives |
|------|--------------|-----------------|
| **Free** | Open to all MCP clients | This public repo (`snowdrop-mcp/skills/`) |
| **Premium** | Subscribers only | Private repo (`snowdrop-core/skills/premium/`) |

A public stub is auto-generated for every premium skill. The stub returns `{"status": "payment_required", ...}` on call. **Never edit stubs manually** — they are regenerated by `scripts/build_public_stubs.py` in snowdrop-core.

### Make it Premium if:

- Implements regulatory compliance logic (SEC, CFTC, MiCA, AMF, ASIC, FSA, GDPR)
- Does financial modeling with proprietary scoring (CLO, NAV, waterfall, stress test)
- Generates structured legal or investor documents
- Would give a competitive advantage if cloned
- Does KYC/AML, chain analysis, or smart contract auditing
- Powers enterprise/white-label features

### Keep it Free if:

- Utility or infrastructure helper (retry logic, logging, formatting)
- Thin wrapper around a public API
- Community or goodwill tool
- Documentation or meta tooling

## Rules

- No hardcoded secrets. Use `os.environ.get("VAR_NAME")`.
- Return structured dicts: `{"status": "ok"|"error", "data": {...}, "timestamp": "ISO8601"}`.
- Keep skills focused — one function, one responsibility.
- Test before PR. Broken skills break the whole server.
